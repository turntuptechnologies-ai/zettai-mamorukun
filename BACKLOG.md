# BACKLOG — 次回実装候補

次の自律開発サイクルで実装を検討する機能リスト。

## 実装済み

- [x] **プロジェクト骨格の構築** — v0.1.0 (#1, PR #2)
- [x] **ハートビート & セルフヘルスチェック** — v0.2.0 (#3, PR #4)

## 候補

1. **モジュールシステムの基盤** — 防御モジュールを動的に登録・管理するレジストリの実装。Module トレイトは定義済みなので、ModuleRegistry を追加しデーモンと統合する
2. **イベントバスの実装** — モジュール間でイベントを伝達する tokio::sync::broadcast ベースの非同期チャネル。EventBus スタブを実装に置き換える
3. **ポートスキャン検知モジュール** — 短時間に多数のポートへの接続試行を検知してログに記録する。Module トレイトの初の実装例
4. **SSH ブルートフォース検知モジュール** — /var/log/auth.log を監視し、認証失敗の連続を検知・ログ記録する（fail2ban 的）
5. **設定ホットリロード** — SIGHUP で設定ファイルを再読み込みし、モジュールの有効/無効を動的に変更する（SIGHUP 受信のスタブは実装済み）
6. **アクションエンジン** — 検知イベントに対するアクション（ログ・通知・ブロック）を設定ベースで実行するエンジン
7. **ログファイル監視フレームワーク** — 任意のログファイルを tail -f 的に監視し、パターンマッチでイベントを発行する汎用フレームワーク
8. **CLI ステータスコマンド** — Unix ソケット経由でデーモンのステータス（稼働時間、メモリ使用量、ロード済みモジュール一覧）を取得するサブコマンド。HealthChecker の情報を活用
9. **IP ブロックリスト管理** — 検知した攻撃元 IP をブロックリストに追加し、nftables 連携でブロックする
10. **メトリクスエクスポーター** — Prometheus 互換のメトリクスエンドポイントを提供し、ハートビート情報・検知統計を外部監視システムに公開する
