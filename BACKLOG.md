# BACKLOG — 次回実装候補

次の自律開発サイクルで実装を検討する機能リスト。

## 候補

1. **プロジェクト骨格の構築** — デーモンとして起動・終了できる最小限の実装。シグナルハンドリング（SIGTERM/SIGINT）、設定ファイル読み込み（TOML）、構造化ログ出力（tracing）
2. **モジュールシステムの基盤** — 防御モジュールを動的に登録・管理するトレイトとレジストリの実装
3. **イベントバスの実装** — モジュール間でイベントを伝達する非同期チャネルベースの仕組み
4. **ポートスキャン検知モジュール** — 短時間に多数のポートへの接続試行を検知してログに記録する
5. **SSH ブルートフォース検知モジュール** — /var/log/auth.log を監視し、認証失敗の連続をブロックする（fail2ban 的）
6. **IP ブロックリスト管理** — 検知した攻撃元 IP をブロックリストに追加し、iptables/nftables 連携でブロックする
7. **設定ホットリロード** — SIGHUP で設定ファイルを再読み込みし、モジュールの有効/無効を動的に変更する
8. **ヘルスチェックエンドポイント** — Unix ソケットまたは HTTP でデーモンのステータスを取得できるエンドポイント
9. **systemd unit ファイルと install サブコマンド** — インストール・アンインストールを簡単にするサブコマンド
10. **統計情報ダッシュボード（CLI）** — ブロック数・検知数等の統計をターミナルに表示するサブコマンド
