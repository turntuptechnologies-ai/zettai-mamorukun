# BACKLOG — 次回実装候補

次の自律開発サイクルで実装を検討する機能リスト。

## 実装済み

- [x] **プロジェクト骨格の構築** — v0.1.0 (#1, PR #2)
- [x] **ハートビート & セルフヘルスチェック** — v0.2.0 (#3, PR #4)
- [x] **ファイル整合性監視モジュール** — v0.3.0 (#5, PR #6)
- [x] **プロセス異常検知モジュール** — v0.4.0 (#7, PR #8)

## 候補

1. **イベントバスの実装** — モジュール間でイベントを伝達する tokio::sync::broadcast ベースの非同期チャネル。EventBus スタブを実装に置き換え、モジュールが検知イベントを発行できるようにする
2. **モジュールレジストリの実装** — 防御モジュールを動的に登録・管理する ModuleRegistry の実装。daemon.rs のモジュール初期化ロジックを汎用化し、新モジュール追加時のボイラープレートを削減する
3. **SSH ブルートフォース検知モジュール** — /var/log/auth.log を監視し、認証失敗の連続を検知・ログ記録する（fail2ban 的）。ログファイル監視の基盤としても機能する
4. **設定ホットリロード** — SIGHUP で設定ファイルを再読み込みし、モジュールの有効/無効を動的に変更する（SIGHUP 受信のスタブは実装済み）
5. **アクションエンジン** — 検知イベントに対するアクション（ログ・通知・ブロック）を設定ベースで実行するエンジン。イベントバスと連携し、検知から対応までのパイプラインを構築する
6. **CLI ステータスコマンド** — Unix ソケット経由でデーモンのステータス（稼働時間、メモリ使用量、ロード済みモジュール一覧、検知統計）を取得するサブコマンド
7. **ファイル整合性監視の拡張** — ベースラインの永続化（JSON/SQLite）、ファイルパーミッション監視、inotify によるリアルタイム検知
8. **プロセス監視の拡張** — プロセスの親子関係チェック、コマンドライン引数の検査、CPU/メモリ異常消費の検知
9. **ネットワーク接続監視モジュール** — /proc/net/tcp, /proc/net/udp を監視し、不審な外部接続（既知の C2 ポート、異常な接続数）を検知する
10. **メトリクスエクスポーター** — Prometheus 互換のメトリクスエンドポイントを提供し、ハートビート情報・検知統計を外部監視システムに公開する
